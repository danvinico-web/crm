# LeadHub — CRM для аффилейт-команды

Приём лидов из внешних источников, отгрузка в чужие CRM («офисы») по API-ключам,
синхронизация статусов лидов в реальном времени и аналитика по офисам и периодам.
Интерфейс 1:1 с макетом (тёмная/светлая тема).

## Стек

- **Next.js 14** (App Router) + **TypeScript** + **Tailwind CSS**
- **MongoDB** + **Mongoose**
- **NextAuth** (credentials: логин + пароль), роли **ADMIN / USER**
- **lucide-react** — иконки
- Шифрование данных at rest — **AES-256-GCM**; «слепые индексы» (HMAC) для поиска/дедупа PII

## Запуск

Нужны Node.js 20+ и npm. MongoDB устанавливать НЕ обязательно — для разработки
поднимается встроенный in-memory MongoDB и база засевается демо-данными.

```bash
npm install
cp .env.example .env.local        # заполните секреты (см. ниже)
npm run dev                       # http://localhost:3000
```

### Секреты для `.env.local`

```bash
# сгенерировать три ключа:
openssl rand -base64 32   # → NEXTAUTH_SECRET
openssl rand -base64 32   # → ENCRYPTION_KEY   (ровно 32 байта)
openssl rand -base64 32   # → BLIND_INDEX_KEY
```

`MONGODB_URI` оставьте пустым для in-memory режима; для прод укажите строку Atlas
или своего сервера. Полный список переменных — в `.env.example`.

## Роли и доступ

- **ADMIN** — главный админ: создаёт пользователей (`/users`), видит всё.
- **USER** — создаёт команды (`/teams`) и агентов, работает с лидами и отправкой.
- **Агент** — доменная сущность (член команды), не аккаунт входа.

Страницы защищены middleware (редирект на `/login`); API проверяют сессию/роль,
вебхуки — HMAC.

## Безопасность

- API-ключи офисов, секреты вебхуков и PII лидов (имя/email/телефон/raw) —
  шифруются в БД (`AES-256-GCM`, ключ `ENCRYPTION_KEY`). В открытом виде и в логи
  не попадают.
- Поиск и дедуп по зашифрованным email/телефон — через HMAC «слепые индексы»
  (`BLIND_INDEX_KEY`), без расшифровки.
- Пароли — bcrypt. Сессии — JWT (NextAuth).
- Входящие вебхуки (приём/статусы) — проверка HMAC-подписи (в фазах приёма/статусов).
